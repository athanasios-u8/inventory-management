Factory Inventory Management — System Architecture

Full-stack application with Vue 3 frontend, Python FastAPI backend, and in-memory mock data.

Tech Stack

Frontend

Vue 3

  • Composition API
  • Vite (port 3000)
  • Vue Router
  • Axios (HTTP client)
  • Custom SVG charts
Backend

FastAPI

  • Python 3.x
  • Pydantic v2 models
  • uv package manager
  • Port 8001
  • CORS enabled
Data

In-Memory JSON

  • 7 JSON data files
  • Loaded at startup
  • No database
  • ~30 inventory items
  • ~200+ orders
State

Composables

  • useFilters (singleton)
  • useI18n (en / ja)
  • useAuth (user session)
  • No external store
Testing

pytest + TestClient

  • FastAPI TestClient
  • Backend API tests
  • tests/backend/

Data Flow

User Interaction

FilterBar Component

User selects Time Period, Warehouse, Category, or Order Status. Each filter is a reactive ref() inside the useFilters() composable (singleton — shared across all views).

Composable

useFilters() — getCurrentFilters()

Views watch() filter refs. On change, getCurrentFilters() maps UI names to API params: { warehouse, category, status, month }. Passed to api.js.

API Client

api.js (Axios)

Centralised HTTP client. Builds URLSearchParams from filters (skips "all" values). Sends GET requests to http://localhost:8001/api/....

Backend

FastAPI Endpoint

Receives query params. Calls apply_filters() sequentially on in-memory data (warehouse → category → status → month/quarter). Returns Pydantic-validated JSON list.

Data Layer

mock_data.py — In-Memory Store

All JSON files loaded once at startup into Python dicts/lists. No DB queries. spending.json has nested structure (summary, monthly, categories). Quarter filter: Q1-2025 expands to Jan/Feb/Mar.

Render

Vue View — Reactive Render

Response assigned to data.value. computed() properties derive totals, percentages, and chart data. Template re-renders. Loading and error states handled in every view.

Filter System

Time Period — month (2025-01) or quarter (Q1-2025) — orders & dashboard only
Warehouse — San Francisco, London, Tokyo — inventory & orders
Category — Circuit Boards, Sensors, Actuators, Controllers, Power Supplies — case-insensitive
Order Status — Delivered, Shipped, Processing, Backordered — orders only

API Endpoints

Endpoint Filters Returns
GET /api/inventory warehouse category List of inventory items with stock levels
GET /api/inventory/{id} — Single inventory item or 404
GET /api/orders warehouse category status month List of orders with items and status
GET /api/orders/{id} — Single order or 404
GET /api/dashboard/summary warehouse category status month KPI summary (inventory value, low stock, pending orders, backlog)
GET /api/demand — Demand forecasts with trend indicators
GET /api/backlog — Shortage items with purchase order status flag
GET /api/spending/summary — Procurement, operational, labor, overhead totals + % change
GET /api/spending/monthly — Monthly breakdown by cost category (Jan–Dec)
GET /api/spending/categories — Spending totals by product category
GET /api/spending/transactions — Recent financial transactions
GET /api/reports/quarterly — Quarterly metrics and fulfillment rates
GET /api/reports/monthly-trends — Month-over-month trend data

Frontend Views

Dashboard

/ — Dashboard.vue

KPI summary cards, order status donut chart, inventory by category bar chart, backlog table, top products. All 4 filters active.

Inventory

/inventory — Inventory.vue

Stock level tracking with search, color-coded status badges (OK / Low / Critical). Filters: warehouse, category. No time dimension.

Orders

/orders — Orders.vue

Order lifecycle with status summary cards and expandable item lists. All 4 filters active.

Demand

/demand — Demand.vue

Demand forecasting with trend badges (increasing / decreasing / stable). No filters — full dataset.

Spending

/spending — Spending.vue

Financial analytics: revenue vs cost, cost flow breakdown (procurement, operational, labor, overhead), monthly trends.

Reports

/reports — Reports.vue

Quarterly business review: quarterly metrics, month-over-month trends, fulfillment rates.
